# Roadmap

## 1. 採点結果のJSON出力

CLI表示とは別に、Web GUIやCIから読みやすいJSON形式の結果を出力できるようにする。

## 2. レベル追加コマンド

`python3 -m app.new_level level_03` のようなコマンドで、`meta.json` や `template.c` まで含む雛形を生成する。

## 3. WA時の差分表示改善

期待値と実際の出力の差を、行単位で初心者にも読みやすく表示する。

## 4. 入力プレビュー

`nodist level <id>` でサンプル入力と期待出力の一部を表示できるようにする。

## 5. レベルタグと難易度

`meta.json` に `tags` と `difficulty` を追加し、探索、配列、ソートなどで分類する。

## 6. コンパイラ設定

`gcc` / `clang` の選択、タイムリミット、追加コンパイルオプションを設定ファイルまたはCLI引数で変更できるようにする。

## 7. GitHub Actionsでの動作確認

Linux環境でCLI、正解例、互換用 `src/judge.py` が動くことをCIで自動確認する。

## 8. GitHub Pages向けデータ公開

`levels/` のMarkdown、JSON、テンプレートを静的サイトから読み込めるよう、公開用インデックスJSONを生成する。

## 9. ブラウザ内エディタの試作

Monaco Editorなどを使い、問題文を見ながらCコードを書けるWeb画面を試作する。

## 10. 教員向け運用ガイド

授業での配布方法、レベル解放の扱い、課題化の方法、採点結果の確認方法、セキュリティ注意をまとめる。
