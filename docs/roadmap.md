# Roadmap

## 1. 採点結果のJSON出力

CLI表示とは別に、Web GUIやCIから読みやすいJSON形式の結果を出力できるようにする。

## 2. 問題メタデータの追加

難易度、タグ、前提知識、推奨学習順を `metadata.json` として問題ごとに管理する。

## 3. テストケースの種類分け

サンプル、基本、境界値、発展のようにテストケースを分類し、学習者が失敗原因を把握しやすくする。

## 4. 入出力差分の見やすい表示

WA時に期待値と実際の出力の差分を、初心者にも読みやすい形で表示する。

## 5. 問題追加用テンプレート生成

`python3 src/new_problem.py <problem_id>` のようなコマンドで問題ディレクトリを作れるようにする。

## 6. clang/gccの選択オプション

利用するコンパイラやコンパイルオプションをCLI引数や設定ファイルで切り替えられるようにする。

## 7. GitHub Actionsでの動作確認

Linux環境で採点器と正解例が動くことをCIで自動確認する。

## 8. GitHub Pages向けデータ公開

問題文Markdown、テストケース、メタデータをWeb GUIから読み込める静的データとして整理する。

## 9. ブラウザ内エディタの試作

Monaco Editorなどを使い、問題文を見ながらCコードを書けるWeb画面を試作する。

## 10. 教員向け運用ガイド

授業での配布方法、課題化の方法、採点結果の確認方法、注意点をまとめたドキュメントを追加する。
