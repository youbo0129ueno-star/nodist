# Curriculum Draft

この教材は、配列上の探索練習ではなく、C言語でデータ構造を実装するための練習教材として設計する。

中心に置くもの:

- `struct` によるセルやリストの表現
- ポインタによる接続関係の管理
- `malloc` / `free` による動的メモリ確保と解放
- `head`, `prev`, `next`, `NIL` の意味
- 操作ごとのポインタ更新箇所
- 計算量の根拠

## Policy

講義ノートでは配列実装も扱うが、C演習の初期レベルでは配列を主役にしない。

まずは構造体とポインタでリストを実装する。配列による疑似ポインタ実装や `freeL` は、動的メモリ版の考え方を理解した後の発展レベルとして扱う。

## Chapter: リスト構造

この章の教材目標:

1. リストは物理的な連続領域ではなく、ポインタのつながりで順序を表すことを理解する
2. 単方向リストと双方向リストの違いを理解する
3. 探索は `head` から `next` をたどる操作だと理解する
4. 挿入は前後の接続をつなぎ直す操作だと理解する
5. 削除は対象セルの前後を直接つなぐ操作だと理解する
6. `malloc` したセルを最後に `free` する必要があることを理解する
7. 操作ごとの計算量を、走査回数とポインタ更新本数から説明できるようにする

## Proposed Level Split

### level_00: Node構造体を定義する

目的:

- `struct Node` を書く
- `key` と `next` を持つセルを理解する

想定する要件:

- `struct Node` を定義する
- `int key`
- `struct Node *next`
- 1個のノードを作り、値を出力する

### level_01: mallocで1個のノードを作る

目的:

- 動的メモリ確保の入口を作る
- `->` 演算子に慣れる

想定する要件:

- `malloc(sizeof(struct Node))` を使う
- 入力された値を `node->key` に入れる
- `node->next` を `NULL` にする
- 最後に `free(node)` する

### level_02: 2個のノードをnextでつなぐ

目的:

- ポインタで順序を表す感覚を作る

想定する要件:

- 2個のノードを作る
- `first->next = second` にする
- `first` からたどって2個の値を出力する

### level_03: 先頭から末尾まで走査する

目的:

- `while (current != NULL)` の形に慣れる

想定する要件:

- `head` から `next` をたどる
- 各ノードの値を順に出力する
- 末尾の `NULL` で停止する

### level_04: 末尾にノードを追加する

目的:

- 末尾探索と接続更新を分けて考える

想定する要件:

- `head` から末尾まで進む
- 最後のノードの `next` に新ノードをつなぐ
- 結果のリストを出力する

### level_05: 先頭にノードを追加する

目的:

- `head` 自体が更新対象になる場合を扱う

想定する要件:

- 新ノードの `next` を古い `head` にする
- `head` を新ノードに更新する
- 結果のリストを出力する

### level_06: リストから値を探す

目的:

- List-Searchを構造体ポインタ版で実装する

想定する要件:

- `head` から順にたどる
- `current->key == target` なら見つかったと判定する
- 見つかれば `FOUND`、なければ `NOT FOUND` を出力する

### level_07: 見つかったノードへのポインタを返す

目的:

- 探索を関数として切り出す

想定する要件:

- `struct Node *list_search(struct Node *head, int target)` を書く
- 見つかればそのノードのポインタを返す
- 見つからなければ `NULL` を返す

### level_08: 指定値のノードを削除する

目的:

- 単方向リストの削除で「直前のノード」が必要になることを理解する

想定する要件:

- `prev` と `current` を使って探索する
- 削除対象が先頭の場合と途中の場合を分ける
- 削除したノードを `free` する

### level_09: リスト全体を解放する

目的:

- 動的に確保した全ノードを安全に解放する

想定する要件:

- `next` を保存してから現在ノードを `free` する
- 最後まで解放する
- 解放後に `head = NULL` と考えられる形にする

### level_10: 双方向リストのNodeを定義する

目的:

- `prev` があることで削除や逆方向移動が楽になることを理解する

想定する要件:

- `struct DNode`
- `int key`
- `struct DNode *prev`
- `struct DNode *next`

### level_11: 双方向リストへ途中挿入する

目的:

- 挿入時に更新する4箇所を理解する

想定する要件:

- `p` の次に `x` を挿入する
- `q = p->next` を使う
- `x->prev`, `x->next`, `p->next`, `q->prev` を正しく更新する

### level_12: 双方向リストからノードを削除する

目的:

- 削除時に前後を直接つなぐ操作を理解する

想定する要件:

- `p = x->prev`
- `q = x->next`
- 先頭削除、末尾削除、途中削除を扱う
- `free(x)` する

### level_13: 最大値・最小値を求める

目的:

- リストでは大小関係が埋め込まれていないため全探索が必要だと理解する

想定する要件:

- `list_maximum`
- `list_minimum`
- 返り値は値ではなくノードへのポインタにする

### level_14: 操作の計算量を答える

目的:

- 実装だけでなく、なぜその計算量になるかを説明できるようにする

想定する要件:

- Search: `O(n)`
- Insert after known node: `O(1)`
- Delete known node in doubly linked list: `O(1)`
- Delete by key: `O(n)`
- Maximum / Minimum: `O(n)`

## Later Extension: Array-Based List and freeL

構造体ポインタ版を理解した後、講義ノートの配列実装に対応する発展レベルを追加する。

候補:

- `key[]`, `prev[]`, `next[]` でセルを表す
- `head` と `next[]` だけが本体リストの順序を決めることを確認する
- `NIL = -1` を使う
- `allocate_object` で `freeL` からセルを取り出す
- `free_object` で削除セルを `freeL` に戻す
- 本体リストと未使用セルリストを混同しない

この単元は「配列を使った実装」ではあるが、目的は配列探索ではなく、アドレスを添字で模倣する考え方を理解することである。
